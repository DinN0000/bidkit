# DA (Data Architecture) Domain Context

DA 팀이 SSOT를 작성할 때 적용하는 구조적 패턴과 품질 기준.

---

## 데이터 아키텍처 구성 순서

| 순서 | 파트 | 내용 |
|------|------|------|
| 1 | **데이터 모델 개요** | 전체 엔티티 관계도, 핵심 테이블/엔티티 목록 |
| 2 | **핵심 데이터 구조** | 주요 엔티티별 속성 정의, 타입, 제약조건 |
| 3 | **데이터 흐름** | 시스템 간 데이터 이동 경로, 변환 규칙 |
| 4 | **저장소 설계** | DBMS 선정 근거, 파티셔닝, 아카이빙 전략 |
| 5 | **보안 설계** | 암호화 범위, 접근제어, 개인정보 처리 |

---

## 데이터 구조 정의 패턴

### 엔티티/테이블 명세

```
| 속성명 | 타입 | 필수 | 설명 | 제약조건 |
```

- 모든 엔티티에 PK, FK 관계 명시
- 코드성 컬럼은 코드 테이블과의 관계 명시
- 날짜 필드는 포맷(ISO 8601 등) 명시

### 계층적 데이터 구조

데이터 간 계층 관계가 있을 때는 트리 구조로 표현:

```
상위 개념
├── 하위 개념 A
│   ├── 속성 1 (설명)
│   ├── 속성 2 (설명)
│   └── 속성 3 (설명)
└── 하위 개념 B
    └── 속성 4 (설명)
```

계층 간 참조 관계(하위→상위)를 명확히 표현.

## 데이터 흐름도 표준

### 참여자 배치 규칙

| 위치 | 대상 |
|------|------|
| 좌측 | 데이터 발생 주체 (사용자, 외부 시스템) |
| 중앙 | 처리 시스템 (WAS, 배치, ETL) |
| 우측 | 데이터 저장소 (DB, 파일, 외부 연동) |

### 상세 흐름 서술 패턴

번호를 붙여 순차적으로 기술하되, 각 단계에 **데이터 변환/검증 규칙**을 명시:

```
① 데이터 수신
  - 수신 포맷, 검증 규칙
② 데이터 변환
  - 변환 로직, 매핑 규칙
③ 데이터 저장
  - 저장소, 트랜잭션 범위
④ 결과 반환
  - 응답 포맷, 오류 코드
```

## 검증룰(Validation Rule) 설계

데이터 무결성을 위한 검증룰은 명시적으로 정의:

| 항목 | 내용 |
|------|------|
| 검증 대상 | 어떤 데이터 항목을 검증하는가 |
| 검증 조건 | 필수값, 범위, 포맷, 참조 무결성 |
| 실패 시 동작 | 거부, 경고, 로깅 |
| 변경 이력 | 검증룰 변경 시 영향도 분석 |

검증룰 변경은 영향받는 화면/프로세스 목록과 함께 기술.

## 관리 화면 데이터 패턴

관리자 화면의 데이터 조회/관리 기능은 다음을 포함:

### CRUD 화면 표준 요소

| 기능 | 필수 요소 |
|------|---------|
| 목록 조회 | 검색 필터, 정렬, 페이징 |
| 상세 조회 | 전체 필드 표시, 감사 이력(생성일/수정일) |
| 등록 | 입력 검증, 중복 체크, 확인 메시지 |
| 수정 | 변경 전/후 비교, 변경 사유 |
| 삭제 | 삭제 확인, 참조 무결성 체크, 논리 삭제 우선 |

### 통계 화면 표준 요소

- 기간 필터 (년/월/일)
- 카테고리별 분류 (업무별, 상태별)
- 합계행 포함
- 엑셀 다운로드
- 페이징

## 데이터 보안 설계

### 암호화 적용 기준

| 구분 | 대상 | 방식 | 키 관리 |
|------|------|------|--------|
| 전송 중 | 외부 통신 | TLS 1.2+ | 서버 인증서 |
| 저장 시 | 개인정보, 인증정보 | AES-256 또는 국가 인증 암호화 | HSM 또는 전용 키관리 |
| 로그 | 민감 데이터 마스킹 | 부분 마스킹 | - |

### 개인정보 처리 원칙

- **최소 수집**: 업무에 필요한 최소한의 정보만
- **목적 외 이용 금지**: 수집 목적 이외 사용 불가
- **파기 규칙**: 보유기간 명시, 목적 달성 후 파기
- **접근 로그**: 개인정보 접근 이력 기록 및 감사

## 마이그레이션 데이터 설계

기존 시스템에서 데이터 이관 시:

| 항목 | 내용 |
|------|------|
| 이관 대상 | 테이블/엔티티별 이관 여부, 건수 |
| 매핑 규칙 | 원본→대상 필드 매핑, 변환 로직 |
| 검증 방법 | 건수 대사, 샘플링 검증, 전수 검증 |
| 롤백 | 이관 실패 시 복구 절차 |

## Critic 검증 포인트 (DA)

| 검증 항목 | Critical 조건 |
|---------|-------------|
| 화면-모델 불일치 | 화면 필드가 데이터 모델에 없음 |
| 암호화 누락 | 개인정보/인증정보 암호화 미적용 |
| 참조 무결성 | FK 관계 미정의 또는 불일치 |
| 검증룰 누락 | 핵심 입력에 검증 규칙 없음 |
| 통계 정합성 | 통계 수치의 합계 불일치 |
| 파기 규칙 미정의 | 개인정보 보유기간/파기 절차 없음 |
| 이관 검증 누락 | 마이그레이션에 검증 절차 없음 |
