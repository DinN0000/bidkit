# Proposal Harness Evals

Lightweight regression fixtures for the highest-risk user flows:

- `/design` kickoff guidance
- `/write` section drafting and exploration
- `/verify` decision-first review output

These are prompt and expected-output fixtures, not a full runner. They exist so
the harness can be checked whenever wording, schemas, or command contracts change.

Minimum expectation for every eval:

1. A user-facing situation label appears early
2. One focused next action is recommended
3. Internal role names do not dominate the first response
4. Proposal Guide appears at the bottom
